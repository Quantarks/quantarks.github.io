The Voice APIs provide enhanced user experience in the Quantarks App by providing audio content that explain topics in greater detail. For more information about how audio content is handled in the app, check out How Audio Content Should Work in the App.

Operations

Getting Audio Files
GET /api/v1/voice/

Returns a *.qmpz file.

PARAMETERS:

auth: string Required
The authentication ID of the app. This is a string saved internally both on the server and the app's configuration. If the auth ID saved in the app and the one in the server does not match, the server will return a 403 Forbidden status code.
keys: string Required
The keys of audio files requested. This should be a stringified JSON array of strings in this format: 
["key1", "key2", "key3", ...]
If this parameter contains invalid JSON, the server returns a 400 Bad Request status code.

RETURNS: A .qmpz file containing the requested audio and transcript files.

Methods

Extract(Uri)
public static QAudio Extract(Uri filePath)
    throws FileNotFoundException, UriFormatException;

Extracts the audio and transcript from a *.qmp3 file.

NOTE: This mereley creates a File object out of the file filePath points to. Then it passes the File object to Extract(File).

PARAMETERS:

  • Uri filePath

    The path of the *.qmp3 file to be extracted.

RETURNS: The new QAudio object that results from this operation.

THROWS:

  • FileNotFoundException

    Thrown when the app cannot find the file pertained to by the URI.

  • UriFormatException

    Thrown when the URI passed to this method has invalid format.
