QUpdateManager is the class that manages updates. It is created to organize updates and eliminate the need for rewriting update logic for every single asset type. Reading how the update sequence should work before starting development will be helpful.

Properties

UpdateURI
public Uri UpdateURI;

The server URL where this object will check for updates.

Methods

FetchUpdateChanges()
public abstract void FetchUpdateChanges() throws Exception;

Downloads the updates.json from the server. It will place the file inside the temp/ folder inside the app's cache folder.

THROWS: an Exception if the server returned an error, or the connection was interrupted.

IsUpdatable()
public abstract bool IsUpdatable() throws FileNotFoundException;

Checks if the temp/updates.json is later than what is currently used by the app. This works by comparing updates.json and assets.json on their update_hash and update_timestamp fields.

THROWS: A FileNotFoundException if either assets.json or temp/updates.json cannot be found.

PrepareAssetList()
public abstract QUpdate PrepareAssetList() throws FileNotFoundException;

Serializes temp/updates.json into a QUpdate object.

RETURNS: a QUpdate object that was created from the updates.json

THROWS: A FileNotFoundException if temp/updates.json cannot be found.
